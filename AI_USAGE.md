## 2. Architecture & Planning Phase

### What AI Helped With:
- **Project structure design**: Recommended the complete folder structure (`pages/api/`, `components/`, `lib/`, etc.)
- **Database schema**: Designed the Prisma schema with `Notice` model, enums for `Category` and `Priority`, and all required fields
- **API route planning**: Planned the 5 API endpoints with correct HTTP methods:
  - `GET /api/notices` - List all
  - `POST /api/notices` - Create
  - `GET /api/notices/[id]` - Read single
  - `PUT /api/notices/[id]` - Update
  - `DELETE /api/notices/[id]` - Delete
- **Tech stack validation**: Confirmed Next.js Pages Router, Prisma, PostgreSQL, and Tailwind CSS met all assignment requirements
- **Deployment strategy**: Planned local PostgreSQL → Neon cloud → Vercel deployment flow

### Human Actions:
- Approved the architecture
- Provided PostgreSQL credentials
- Set up Neon account and provided connection string
- Verified all requirements against assignment PDF

## 4. Debugging & Troubleshooting

### Issues AI Helped Resolve:

1. **Prisma Configuration Conflict**
   - Problem: `prisma.config.ts` in parent directory was overriding project settings
   - Solution: AI suggested deleting the conflicting config file and using direct schema path

2. **PowerShell Command Syntax**
   - Problem: `&&` operator not supported in PowerShell
   - Solution: AI provided `cmd /c` wrapper for multi-command execution

3. **TypeScript Errors in NoticeForm**
   - Problem: Implicit `any` types in form inputs
   - Solution: AI added proper TypeScript interfaces and generic types

4. **Next.js Build Warning**
   - Problem: Multiple lockfiles causing workspace root detection warning
   - Solution: AI added `turbopack.root` config to `next.config.ts`

5. **Image Upload Implementation**
   - Problem: File upload with formidable caused type errors
   - Solution: AI suggested dropping complex upload and using URL field (simpler for assignment)

6. **Date Validation Requirement**
   - Problem: PDF said publishDate is not required but form was validating it
   - Solution: AI removed the required validation from both client and server

7. **Vercel CLI Authentication**
   - Problem: Scope errors and interactive prompts blocking deployment
   - Solution: AI suggested using Vercel dashboard instead of CLI for env variables

## 6. Database Migration Guidance

### Local PostgreSQL Setup:
- AI provided connection string format for local `.env`
- Guided creation of `notice_board` database
- Assisted with `npx prisma migrate dev` command

### Neon Cloud Migration:
- Explained why Neon was recommended (free tier, PostgreSQL compatible, no credit card)
- Provided step-by-step Neon setup instructions
- Helped update `.env` with Neon connection string
- Guided migration command to apply schema to cloud DB

### Connection String Switch:
AI suggested a clean `.env` pattern for switching between local and cloud:
```env
# Local (development)
# DATABASE_URL="postgresql://postgres:password@localhost:5432/notice_board"

# Cloud (production)
DATABASE_URL="postgresql://neondb_owner:password@ep-xxx.neon.tech/neondb?sslmode=require"
```
